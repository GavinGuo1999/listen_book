# 2026-06-22 Progress

## 今天完成

- 确定书库产品方向：
  - 公共书库为主。
  - 普通用户上传需要管理员审批。
  - EPUB 解析当前够用，暂不继续推进 PDF。

- 新增书籍审核/发布最小闭环。
  - 新增 `Book.uploader_id`、`Book.review_status`、`Book.review_note`。
  - 新增 Alembic 迁移：`backend/alembic/versions/20260622_0002_book_review_workflow.py`。
  - 老书迁移后默认 `approved`，避免现有书库消失。
  - 管理员上传自动 `approved`。
  - 普通用户上传默认 `pending_review`。
  - 管理员可通过 `PATCH /api/books/{book_id}/review` 批准或拒绝。

- 新增公共书库可见性规则。
  - 管理员可见全部书籍。
  - 普通用户可见已批准公共书籍，以及自己上传的待审/拒绝书籍。
  - 其他普通用户不能看到未审批书籍。
  - 章节、阅读进度、音频生成/预取/状态/文件下载都会检查书籍可见性。

- 收紧删除权限。
  - 管理员可以删除任意书籍。
  - 普通用户只能删除自己上传且尚未批准的书籍。

- 收敛上传格式。
  - 当前支持 TXT/EPUB。
  - PDF 暂不接入，避免上传后解析失败的假支持。

- 管理员初始化。
  - 第一个正式注册用户自动成为管理员。
  - 默认本地用户 `local` 仍是管理员，用于本地单机/开发体验。

- 前端新增审批体验。
  - 书籍行显示 `ready/failed/...` 解析状态。
  - 待审/拒绝书籍额外显示“待审批/已拒绝”徽标。
  - 管理员书籍行显示“批准/拒绝”按钮。
  - 补充登录/注册表单的 `data-testid`，方便 E2E。

- 补登录链路浏览器 E2E 脚本。
  - 注册。
  - 退出登录。
  - 重新登录。

- 实机启动验证时修复两个问题。
  - `scripts/smoke_api.py` 的 `httpx` 访问本机服务时会受环境代理影响拿到 502；已改为 `trust_env=False`。
  - 注册成功后退出登录，表单仍停在“注册”页签，导致 E2E 重新登录时误发注册请求并返回 409；已改为注册成功和退出登录后默认切回“登录”页签。
