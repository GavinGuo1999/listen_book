# 2026-06-25 Progress

## 今天完成

- 重做认证入口和登录态边界。
  - 新增独立 `/login` 登录/注册页。
  - 未登录用户不能访问书库、上传、阅读、音频、进度或审批接口。
  - 登录态改为后端写入 `HttpOnly` Cookie：`listen_book_session`。
  - 前端不再使用 `localStorage` 保存 token。
  - 后端不再给无认证请求回退到 `local` 默认用户。

- 注册表单去掉“显示名”。
  - 注册只需要用户名和密码。
  - 后端默认 `display_name = username`。

- 管理员入口和初始化逻辑改造。
  - `/admin` 作为独立管理员后台入口。
  - 普通业务页不再混入审批中心。
  - 删除“第一个注册用户自动成为管理员”的规则。
  - 新增 `.env` 显式 bootstrap 管理员：
    - `LISTEN_BOOK_BOOTSTRAP_ADMIN_USERNAME`
    - `LISTEN_BOOK_BOOTSTRAP_ADMIN_PASSWORD`
  - 服务启动时会确保该用户存在、启用、拥有管理员权限，并同步配置里的密码。

- 清理开发库污染数据。
  - 新增 `scripts/reset_dev_data.py`。
  - 已清理开发库里的测试用户、旧 `local` 用户、书籍、章节、句子、音频、任务、审批记录和 storage 文件。
  - 清理后确认库里至少有 bootstrap 管理员 `admin`。
  - 后续如果要重置开发库，可再次运行该脚本。

- 改善认证错误提示。
  - 后端不再统一返回 `Invalid credentials`。
  - 现在明确区分：
    - `请输入用户名`
    - `请输入密码`
    - `用户名至少需要 3 个字符`
    - `密码至少需要 6 个字符`
    - `用户名不存在`
    - `密码不正确`
    - `用户名已存在`
    - `用户已停用`
  - 前端会解析后端 JSON 的 `detail`，不再直接显示 `{"detail":"..."}`。
  - 前端登录/注册表单也做了基础校验。

- 更新测试和文档。
  - 后端认证测试覆盖登录/注册边界错误。
  - E2E 文件新增认证错误提示用例。
  - README 和 runbook 已同步新管理员规则。

- 管理员账号通过 `.env` bootstrap，默认用户名是 `admin`。

- 普通注册用户不会成为管理员。
